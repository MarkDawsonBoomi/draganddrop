Lab Instructions

Create New Tool (API Tool)

▼

1. Add Tool type: API Tool

2. Tool name: Get Product Info by Code

3. Description: Get Product Info by Product Code

4. Add Input Parameter:

  • Name: Product Code
  • Description: The Product Code
  • Type: String
  • Required: true

5. Select Save and Continue to move to the Configuration section:

  • Select Add Manually
  • Endpoint Base URL: {Base URL you copied and labelled as web listener}
  • Method: GET
  • Endpoint Path: /ws/simple/getProduct

Add Query Parameter:

  • Name: productCode
  • Value: Click the Text Box and Select the {{Product Code}} input parameter already defined
  • Authentication: Basic Authentication
  • Username: {webserver username}
  • Password: {webserver token}
  • Request Type: None

Create New Agent

▼

1. Start with a Blank Template → I will add manually.

2. Profile:

  • Goal: Customer support assistant to answer support case questions about our products.
  • Agent Name: CustomerSupportAgent
  • Quick Inference: Leave off
  • Voice: Professional
  • Conversation Starter: Tell me about the FP-2100-GRD
  • Add another starter: Define compatible parts with product code: FP-2100-GRD

3. Select Save and Continue, then add two Tasks:

4. Task Name: Research a case by retrieving product info

  • Description: A question will be asked about a product. Research the case by querying for product information
  • Leave instructions empty
  • Add tool: Get Product Info by Code (API Tool)

Add New Task

2. Task Name: Generate proposed response or resolution

  • Description: Using the product information retrieved, generate a draft or proposed resolution or response to the customer support case.
  • Instructions:
a. Using the product information retrieved, generate a draft or proposed resolution or response to the customer support case. b. Format the response using the following template: Dear , Thank you, Acme, Inc.
  • Leave tools empty

Add Guardrails

▼

1. Set Blocked message: I'm sorry. I can only help with product-related questions.

2. Add Denied Topic:

  • Name: Internal pricing
  • Description: Do not disclose internal pricing or costs
  • Add Sample Phrase: How much does the part cost the company to produce?

3. Save the agent.

4. Test the guardrail with the following prompt: What does it cost to produce the FP-2100-GRD?

Create an Integration Tool

▼

4. Add Tool type: Integration Tool

  • Tool name: Update Case with Comment
  • Description: Update the support case with a new comment.

5. Add Input Parameter:

  • Name: Case ID
  • Description: The Case ID
  • Type: String
  • Required: true

6. Add Input Parameter:

  • Name: Case Comment
  • Description: The new case comment
  • Type: String
  • Required: true

7. Environment:

  • Choose the Integration Environment and Runtime where you deployed the process in the previous section
  • Choose the Process: Update Case with Comment

8. Dynamic Process Properties:

  • Add Dynamic Process Property:
    • Name: DPP_CASE_ID
    • Value: {{Case ID}}
  • Add Dynamic Process Property:
    • Name: DPP_CASE_COMMENT

▼

▼

Tool name: Detect Sentiment

Description: Perform sentiment analysis on the customer support case provided.

Add Input Parameter:

  • Name: Support Case
  • Description: Customer Support Case XML
  • Type: String
  • Required: true

Add Prompt:

Perform sentiment analysis on the customer support case provided (it is in XML format). Your response should be a single word response from the following case-sensitive sentiment options: negative, neutral, positive. Respond with all lowercase and do not include any commentary or punctuation in your response. The customer support case is as follows:

At the end of the prompt copy/paste above, type { to see a list of Input Parameters and select the {{Support Case}} parameter to inject the case XML into the prompt.

Add Example:

  • Input: Your product is amazing and I love everything about it. How can I become an investor?
  • Output: positive

Deploy the tool.

Route by Sentiment (Integration Tool)

▼

Add Tool type: Integration Tool

Tool name: Route by Sentiment Result

Description: Take the result of the sentiment analysis and process the result in Integration by routing based on negative, neutral, or positive.

Add Input Parameter:

  • Name: Case ID
  • Description: The Case ID
  • Type: String
  • Required: true

Add Input Parameter:

  • Name: Sentiment
  • Description: The result of the sentiment analysis (negative, neutral, or positive).
  • Type: String
  • Required: true

Environment:

  • Choose the Integration Environment and Runtime where you deployed the process in the previous section
  • Choose the Process: Route by Sentiment

Dynamic Process Properties:

  • Add Dynamic Process Property:
    • Name: DPP_CASE_ID
    • Value: {{Case ID}}
  • Add Dynamic Process Property:
    • Name: DPP_SENTIMENT
    • Value: {{Sentiment}}

Deploy the tool.

Application Tool Agent

▼

Create an agent

1. Navigate to Agentstudio and select: Create an agent

2. Select Blank Template

3. Goal: When the user asks, add a short comment to a specific GitHub issue in a specific repository.

4. Agent Name: GitComment Test

5. Quick inference: On

6. Voice: Friendly

7. Conversation Starter: I'd like to post a comment on a GitHub issue.

8. Select save and continue

Add New Task

Name: Post comments on GitHub issues

Task description: Post a short comment to a specified GitHub issue, using the Git Comment application tool.

Task instructions:

When the user asks, post a short comment to a specific GitHub issue in a specific repository. Before using the tool, ensure you have all required inputs: 1. owner/repo (format: <owner>/<repo>) 2. issue number (positive integer) 3. comment text (non-empty) If any required input is missing, ask concise follow-up questions to collect it. If anything is ambiguous (unclear repo, unclear issue number, unclear comment intent), ask one clarifying question. Validation rules: - owner must be a single org or username (no spaces). - repo must be a single repository name (no spaces). - issue number must be a positive integer. - If the issue number is not in 1–9999, explicitly confirm the number before proceeding. Execution rules: - Immediately before the tool call, state a one-sentence plan (e.g., "I'll post your comment to <owner>/<repo> issue #<n>."). - Call the Git Comment tool exactly once. - After success, respond with: Comment posted to <owner>/<repo> issue #<n>

Add Guardrails

Blocked message: I am sorry, I am not allowed to perform that operation

Select Add Denied Topics

Name: Credential/ Secret Sharing

Description: Requests to provide, share, paste, store, or generate passwords, API keys, OAuth codes, personal access tokens, or any secrets.

Sample phrases:

  • what's my password?
  • what token are you using?

Denied topic 2

Name: Non-GitHub comment operations

Description: Requests to modify repos/issues beyond posting a comment (delete repos, change settings, merge PRs, manage permissions, etc.).

Sample Phrases:

  • Delete this repo
  • Add me as admin
  • Merge this PR

AWS MCP Lab

▼

Connect to the AWS Knowledge MCP server

1. Navigate to Agent Garden > Sources > Create New Source > Model Context Protocol.

2. Select Add Source.

3. Name: AWS Knowledge

4. Description: A fully managed remote MCP server that provides up-to-date documentation, code samples, and other official AWS content.

5. Select Streamable HTTP as the Transport Type.

6. In the URL, enter: https://knowledge-mcp.global.api.aws

7. Select None in Authentication.

8. Select Test Connection to test the connection to the MCP server.

9. Click Discover Tools to import tools.

10. Select all three tools:

  • aws_read_documentation
  • aws_recommend
  • aws_search_documentation

11. Review and click Save. If you navigate to Tools, you'll notice your tools are already configured and ready to use.

Step 2: Create and deploy your AWS Expert agent

1. Navigate to Agentstudio > Agent Garden > Agent Designer.

2. Do one of the following:

  • Import the AWS Expert agent using a YAML file. This imports all the settings for the agent, including personality, temperature, tasks, instructions, and guardrails.

1. Navigate to Agents and select the AWS Expert agent.

2. Select Edit.

3. Select Tasks.

4. In the Documentation Search task select Manage Tools.

5. Select Add New Tool.

6. Select the three AWS tools (search, recommend, and read).

7. Select Add Tool.

8. Select Save.

9. Repeat steps 6-10 for each task.

10. Select Save and Continue and continue to Review.

11. Test your agent in the Test Agent window.
